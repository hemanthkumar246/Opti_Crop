# OptiCrop &mdash; Smart Agricultural Production Optimization Engine

An end-to-end machine learning web application that recommends the most
suitable crop for a plot of land based on soil nutrients (Nitrogen,
Phosphorous, Potassium) and climate conditions (temperature, humidity,
pH, rainfall).

## Project structure

```

│   └── static/
│       └── style.css
├── requirements.txt
└── README.md
```

## How it works

1. **Data collection & EDA** &mdash; `data/Crop_recommendation.csv` holds soil
   and climate readings for 22 crops. `model/train_model.py` performs
   univariate, bivariate, and multivariate analysis, checks for null
   values, and removes outliers from the phosphorous/potassium columns
   using the interquartile range (IQR) method.
2. **Pattern discovery** &mdash; K-Means clustering (with the elbow method
   to pick the number of clusters) groups similar agricultural
   conditions together.
3. **Model training** &mdash; A `LogisticRegression` classifier is trained
   on the 7 features (nitrogen, phosphorous, potassium, temperature,
   humidity, ph, rainfall) to predict the crop label, then saved to
   `model/model.pkl` with `pickle`.
4. **Web application** &mdash; A Flask app (`app/app.py`) loads `model.pkl`
   and serves three pages:
   - **Home** &mdash; project overview
   - **About** &mdash; how the recommendation pipeline works
   - **Find your crop** &mdash; a form where a user enters their soil/climate
     readings and receives an instant crop recommendation

## Setup

```bash
# 1. Create and activate a virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate        # on Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt
```

## Regenerating the dataset and model (optional)

The trained `model/model.pkl` is already included, so this step is
optional unless you want to retrain from scratch or inspect the EDA
plots.

```bash
python3 data/generate_dataset.py   # (re)creates data/Crop_recommendation.csv
python3 model/train_model.py       # runs the full pipeline and re-saves model.pkl
```

Training prints dataset shape, null-value checks, seasonal crop
groupings, cluster membership, and a full classification report, and
saves EDA charts + the report to `model/plots/`.

## Running the web app

```bash
cd app
python3 app.py
```

Then open **http://127.0.0.1:5000** in your browser. Go to
**Find your crop**, enter values such as:

| Field       | Example |
|-------------|---------|
| Nitrogen    | 90      |
| Phosphorous | 42      |
| Potassium   | 43      |
| Temperature | 20.9    |
| Humidity    | 82      |
| pH          | 6.5     |
| Rainfall    | 202.9   |

and submit to see the recommended crop (this example predicts **rice**).

## Notes on the dataset

The included `Crop_recommendation.csv` is a synthetic dataset generated
to match the structure and agronomic characteristics of the well-known
Kaggle "Smart Agricultural Production Optimizing Engine" crop
recommendation dataset (N, P, K, temperature, humidity, ph, rainfall,
label across 22 crops). If you have access to the original dataset from
Kaggle, you can drop it in at `data/Crop_recommendation.csv` (same
column names) and re-run `model/train_model.py` to retrain on it
directly.
